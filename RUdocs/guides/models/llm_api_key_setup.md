# Настройка API ключа модели

Для взаимодействия RAGFlow с онлайн AI моделью требуется API ключ. В этом руководстве представлена информация о том, как установить API ключ модели в RAGFlow.

## Получение API ключа модели

RAGFlow поддерживает большинство популярных LLM (Large Language Models — большие языковые модели). Пожалуйста, ознакомьтесь с [Supported Models](../../references/supported_models.mdx) для полного списка поддерживаемых моделей. Вам необходимо получить API ключ модели через онлайн-запрос. Обратите внимание, что большинство провайдеров LLM предоставляют новым аккаунтам пробный кредит, который истекает через несколько месяцев, либо промо-лимит бесплатного использования.

:::note
Если вы не нашли вашу онлайн LLM в списке, не расстраивайтесь. Список расширяется, и вы можете [отправить запрос на добавление функции](https://github.com/infiniflow/ragflow/issues/new?assignees=&labels=feature+request&projects=&template=feature_request.yml&title=%5BFeature+Request%5D%3A+)! Кроме того, если у вас есть кастомизированные или локально развернутые модели, вы можете [подключить их к RAGFlow с помощью Ollama, Xinference или LocalAI](./deploy_local_llm.mdx).
:::

## Настройка API ключа модели

У вас есть два варианта настройки API ключа модели:

- Настроить его в файле **service_conf.yaml.template** до запуска RAGFlow.
- Настроить его на странице **Model providers** после входа в RAGFlow.

### Настройка API ключа модели до запуска RAGFlow

1. Перейдите в директорию **./docker/ragflow**.
2. Найдите запись **user_default_llm**:
   - Обновите `factory` на выбранную вами LLM.
   - Обновите `api_key` на ваш ключ.
   - Обновьте `base_url`, если вы используете прокси для подключения к удалённому сервису.
3. Перезагрузите систему, чтобы изменения вступили в силу.
4. Войдите в RAGFlow.  
   _После входа в RAGFlow выбранная модель появится в разделе **Added models** на странице **Model providers**._

### Настройка API ключа модели после входа в RAGFlow

:::caution WARNING
После входа в RAGFlow настройка API ключа модели через файл **service_conf.yaml.template** перестанет работать.
:::

После входа в RAGFlow вы можете *только* настроить API ключ на странице **Model providers**:

1. Нажмите на ваш логотип в правом верхнем углу страницы **>** **Model providers**.
2. Найдите карточку вашей модели в разделе **Models to be added** и нажмите **Add the model**.
3. Вставьте ваш API ключ модели.
4. Укажите базовый URL, если вы используете прокси для подключения к удалённому сервису.
5. Нажмите **OK** для подтверждения изменений.
