# Компонент Token chunker (токен-чанкер)

Компонент, который разбивает тексты на чанки (фрагменты), соблюдая максимальный лимит токенов и используя разделители для поиска оптимальных точек разрыва.

---

Компонент **Token chunker** — это сплиттер текста, который создает чанки, соблюдая рекомендуемую максимальную длину в токенах, используя разделители для обеспечения логичных точек разрыва чанков. Он разбивает длинные тексты на подходящего размера, семантически связанные чанки.

## Сценарий

Компонент **Token chunker** является опциональным, обычно размещается сразу после **Parser** или **Title chunker**.

## Конфигурации

### Рекомендуемый размер чанка

Рекомендуемый максимальный лимит токенов для каждого создаваемого чанка. Компонент **Token chunker** создает чанки в указанных разделителях. Если лимит токенов достигается до разделителя, чанк создается в этой точке.

### Процент перекрытия (%)

Определяет процент перекрытия между чанками. Подходящая степень перекрытия обеспечивает семантическую связность без создания избыточных, повторяющихся токенов для LLM.

- По умолчанию: 0
- Максимум: 30%

### Разделители

По умолчанию `\n`. Нажмите кнопку **Recycle bin** справа, чтобы удалить разделитель, или нажмите **+ Add**, чтобы добавить новый разделитель.

### Вывод

Глобальное имя переменной для вывода компонента **Token chunker**, которое может быть использовано последующими компонентами в пайплайне загрузки данных.

- По умолчанию: `chunks`
- Тип: `Array<Object>`
