# Компонент Switch

Компонент, который оценивает, выполнены ли заданные условия, и направляет поток выполнения соответственно.

---

**Компонент Switch** оценивает условия на основе вывода конкретных компонентов, направляя поток выполнения для реализации сложной ветвящейся логики.

## Сценарии

**Компонент Switch** необходим для условного направления потока выполнения. Хотя он имеет сходство с компонентом [Categorize](./categorize.mdx), который также используется в многонаправленных стратегиях, ключевое отличие заключается в подходе: оценка компонента **Switch** основана на правилах, тогда как компонент **Categorize** использует искусственный интеллект (AI) и применяет LLM для принятия решений.

## Конфигурации

### Случай n

**Компонент Switch** должен иметь как минимум один случай, каждый из которых содержит несколько заданных условий. При указании нескольких условий для случая необходимо установить логическую связь между ними: И (AND) или ИЛИ (OR).

После добавления нового случая перейдите к компоненту **Switch** на холсте, найдите кнопку **+** рядом со случаем и нажмите её, чтобы указать последующий(ие) компонент(ы).


#### Условие

Оценивает, соответствует ли вывод конкретных компонентов заданным условиям

:::danger ВАЖНО
При добавлении нескольких условий для конкретного случая появляется поле **Логический оператор**, в котором необходимо установить логическую связь между этими условиями как И (AND) или ИЛИ (OR).
:::

- **Оператор**: оператор, необходимый для формирования условного выражения.
  - Равно (по умолчанию)
  - Не равно
  - Больше чем
  - Больше или равно
  - Меньше чем
  - Меньше или равно
  - Содержит
  - Не содержит
  - Начинается с
  - Заканчивается на
  - Пусто
  - Не пусто
- **Значение**: одно значение, которое может быть целым числом, числом с плавающей точкой или строкой.  
  - Разделители, множественные значения или выражения *не* поддерживаются.
