# Компонент Indexer

Компонент, который определяет, как индексируются чанки (фрагменты данных).

---

Компонент **Indexer** индексирует чанки и настраивает их форматы хранения в документном движке.

## Сценарий

Компонент **Indexer** является обязательным завершающим компонентом для всех конвейеров загрузки данных (ingestion pipelines).

## Настройки

### Метод поиска

Этот параметр настраивает, как чанки хранятся в документном движке: в виде полнотекстового поиска, эмбеддингов (embeddings) или обоих вариантов.

### Вес эмбеддинга имени файла

Этот параметр определяет вклад имени файла в итоговый эмбеддинг, который является взвешенной комбинацией содержимого чанка и имени файла. По сути, более высокое значение придаёт имени файла большее влияние в итоговом *составном* эмбеддинге.

- 0.1: Имя файла вносит 10% (содержимое чанка 90%)
- 0.5 (максимум): Имя файла вносит 50% (содержимое чанка 90%)
